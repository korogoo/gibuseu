# 스터디원 가이드

## 발표 등록하는 법

1. GitHub 저장소 → **Issues** → **New Issue**
2. 발표 대분류에 맞는 템플릿 선택 (예: DB 발표면 "발표 등록 - 데이터베이스")
3. **제목에 발표 주제를 바로 입력** (별도 주제 필드 없음)
4. 본문 항목 작성
   - 발표자: **본인 이름을 `members.yaml`에 등록된 것과 정확히 같게** 입력 (다르면 봇이 코멘트로 알려줌)
   - 발표일: `YYYY-MM-DD`
   - 발표 시간: `HH:MM` (회차마다 다를 수 있으니 정확히! 이 시간 기준으로 1시간 전 알림이 갑니다)
   - 소분류: 드롭다운에서 선택, 없으면 "기타" 선택 후 아래 직접입력 칸에 작성
   - 유형: 이론 학습 / 트러블슈팅
   - 학습 완료기준: 자유 서술, `-`로 시작하는 줄 최소 2개 이상
   - 블로그 링크: 발표 끝난 뒤 채워도 됨. 채운다면 `http(s)://`로 시작하는 URL 형식이어야 함

조는 따로 입력하지 않아도 됩니다 — 등록하면 봇이 `teams/history.yaml`의 최신 회차에서 본인 이름을 찾아 자동으로 조 라벨(`1조`/`2조`/`3조`)을 붙여줍니다.

카테고리별 소분류 전체 목록은 [`CATEGORIES.md`](./CATEGORIES.md) 참고 (필수 아님, 드롭다운에 다 있음).

## 등록하면 자동으로 일어나는 일

- 대분류 라벨이 자동으로 붙음
- 작성자 본인이 담당자(assignee)로 자동 지정됨 (레포 협업자로 초대돼 있어야 동작)
- **기부스지키미👀💸** 봇이 발표자 이름/발표일 형식/완료기준 줄 수/소분류-기타 누락/블로그 링크 형식을 검사해서 문제 있으면 코멘트 + `형식 확인 필요` 라벨을 남김 — 코멘트 내용대로 수정하면 라벨은 자동으로 없어짐
- 같은 봇이 조도 자동으로 라벨링함 (위 참고)
- [Projects 보드](https://github.com/users/korogoo/projects/8)에 자동으로 들어감 (발표일 기준 캘린더로 볼 수 있음)
- [README.md](./README.md)의 "발표 진행상황" 표에도 자동으로 반영됨

- 발표 전날 아침 9시·저녁 9시, 아직 등록 안 한 사람이 있으면 디스코드로 알림
- **발표 시작 1시간 전**, 등록된 발표일+시간 기준으로 디스코드에 자동 알림 (15분 간격으로 체크하니 최대 15분 오차 있을 수 있음)
- 발표일이 지났는데 이슈가 안 닫혀있으면 매일 아침 9시 디스코드+이슈 코멘트로 알림 ("취업은 조상님이 시켜주나요? 미루지 말고 블로그 작성하세요👴🏻💸") — 블로그 링크 남기고 이슈 닫을 때까지 계속 옴

## 발표 끝난 뒤

1. 정리한 내용을 본인 블로그에 포스팅
2. 등록했던 Issue에 블로그 링크 코멘트로 남기기
3. Issue 닫기 (Close)
