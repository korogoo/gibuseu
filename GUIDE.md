# 스터디원 가이드

## 발표 등록하는 법

1. GitHub 저장소 → **Issues** → **New Issue**
2. 발표 대분류에 맞는 템플릿 선택 (예: DB 발표면 "발표 등록 - 데이터베이스")
3. **제목에 발표 주제를 바로 입력** (별도 주제 필드 없음)
4. 본문 항목 작성
   - 발표자: **본인 이름을 `members.yaml`에 등록된 것과 정확히 같게** 입력 (다르면 봇이 코멘트로 알려줌)
   - 발표일: `YYYY-MM-DD 23:00` (시간은 매 회차 **23:00 고정**이지만, 날짜 뒤에 시간까지 함께 적어주세요)
   - 소분류: 드롭다운에서 선택, 없으면 "기타" 선택 후 아래 직접입력 칸에 작성
   - 유형: 이론 학습 / 트러블슈팅
   - 학습 완료기준: 자유 서술, `-`로 시작하는 줄 최소 2개 이상
   - 블로그 링크: 발표 끝난 뒤 채워도 됨. 채운다면 `http(s)://`로 시작하는 URL 형식이어야 함

조는 **D-2 자정(주제 제출 마감) 다음날인 D-1**에 제출된 주제들의 연관성(소분류 → 대분류)을 기준으로 자동 배정됩니다 — 등록 시점엔 아직 조가 안 정해져 있는 게 정상이고, D-1이 지나면 이슈에 조 라벨(`1조`/`2조`/`3조`)이 자동으로 붙습니다.

카테고리별 소분류 전체 목록은 [`CATEGORIES.md`](./CATEGORIES.md) 참고 (필수 아님, 드롭다운에 다 있음).

## 등록하면 자동으로 일어나는 일

- 대분류 라벨이 자동으로 붙음
- 작성자 본인이 담당자(assignee)로 자동 지정됨 (레포 협업자로 초대돼 있어야 동작)
- **기부스지키미👀💸** 봇이 발표자 이름/발표일 형식/완료기준 줄 수/소분류-기타 누락/블로그 링크 형식을 검사해서 문제 있으면 코멘트 + `형식 확인 필요` 라벨을 남김 — 코멘트 내용대로 수정하면 라벨은 자동으로 없어짐
- **문제 없이 등록되면 디스코드에 "📢 새 발표 등록!"으로 주제/발표자/일시가 자동 공지됨** — 다른 스터디원들이 무슨 주제인지 바로 알 수 있음
- 등록 후 내용을 수정하면 "✏️ 발표 정보 수정됨" 알림이, 이슈를 삭제하면 "🗑️ 발표 취소/삭제됨" 알림이 감
- [README.md](./README.md)의 "발표 진행상황" 표에도 자동으로 반영됨

## 리마인더

- **D-2(다음 발표일 이틀 전) 자정이 주제 제출 마감**입니다. 마감일 아침 9시·저녁 9시, 아직 주제를 안 낸 사람이 있으면 디스코드로 알림. 마감을 못 지키면 D-1 조 배정 때 가장 작은 조에 강제로 편입됩니다
- **D-1에 연관성 기준으로 조 배정**이 이뤄지고 디스코드로 조 편성이 공지됩니다
- **발표 시작(23:00) 1시간 전**, 디스코드에 자동 알림 (15분 간격으로 체크하니 최대 15분 오차 있을 수 있음)
- 발표일이 지났는데 이슈가 안 닫혀있으면 매일 아침 9시 디스코드+이슈 코멘트로 알림 ("취업은 조상님이 시켜주나요? 미루지 말고 블로그 작성하세요👴🏻💸") — 블로그 링크 남기고 이슈 닫을 때까지 계속 옴

## 발표 끝난 뒤

1. 정리한 내용을 본인 블로그에 포스팅
2. 등록했던 Issue 본문의 "블로그 링크" 항목을 실제 링크로 수정
3. Issue 닫기 (Close) — **블로그 링크 없이 닫으면 봇이 바로 재오픈하고 코멘트를 남깁니다.** 링크 채운 뒤 다시 닫아주세요
4. 정상적으로 닫히면 디스코드에 축하 메시지 + 블로그 링크가 자동 공지됩니다
